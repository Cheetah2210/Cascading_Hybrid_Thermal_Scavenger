# Uncertainty Analysis & Bench Test Methodology

## 1. Sensor Error Margins
To transition from a "theoretical model" to "validated hardware," we define the following sensor jitter and accuracy thresholds:

| Sensor Type | Application | Accuracy/Error Margin |
| :--- | :--- | :--- |
| K-Type Thermocouples | Thermal Gradient | ±1.5°C |
| Mass-Flow Transducers | Fluid Loop Velocity | ±0.5% |
| Precision Data Loggers | Power Balance (V/I) | ±0.1% |

## 2. Statistical Propagation
System uncertainty ($U_{total}$) is calculated via the Root-Sum-Square (RSS) method to ensure statistical independence of variables:

$$U_{total} = \sqrt{(U_{MHD})^2 + (U_{TEG})^2 + (U_{Zeo})^2}$$

## 3. Current Confidence Intervals
Due to the lack of long-term commercial-scale reliability data, all outputs generated by the `GEN_III` node are currently subject to a **±12% experimental uncertainty index**. 

This index is non-static and will be auto-refined as `validation/prototype_telemetry.csv` accumulates steady-state run time.

## 4. Calorimetry Reconciliation
Any variance in energy balance (Input vs. Output + Rejected Heat) exceeding **0.5 kW** flags the telemetry point as "ANOMALOUS," requiring re-calibration of the MHD shunt or TEG interface.
